## EPIC-0003: 可控外部知识摄入 (Controlled Knowledge Ingestion)

## 目标 (Objective)

为 Agent 提供一种安全、确定性且可引用的方式来摄入外部知识 (URLs, Papers, Docs)。解决 Agent 随意访问互联网导致的上下文不可控和幻觉问题。

## 概念: Spike (探针)

Spike 原意为敏捷开发中的“探针任务”。在 Monoco 中，我们将 Spike 定义为**“对外部知识的切片引用”**。

1. **确定性**: 每个 Spike 都是本地 `.references/` 目录下的一个文件，Agent 读取的是固化的本地副本，而非易变的互联网内容。
2. **可引用**: 通过 `[[SPIKE-001]]` 引用知识，就像引用代码一样。

## 关键交付 (Key Deliverables)

1. **Spike CLI (`monoco spike`)**:
   - `add <url>`: 下载 URL 内容，转换为 Markdown，存入 `.references/`。
   - `list`: 列出已有知识。
   - `read <id>`: 读取知识内容。
2. **知识清洗 (Content Cleaning)**: 使用 `trafilatura` 或类似工具提取核心文本，去除广告和噪音。
3. **引用完整性 (Referential Integrity)**: 确保 Issue 中引用的 Spike ID 在本地存在。
